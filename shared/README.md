# Общие ресурсы / Shared

Эта директория содержит материалы, общие для всех практикумов в курсе.

## Содержимое

| Файл | Описание | Для кого |
|------|---------|---------|
| **README.md** | Этот файл | |
| **COURSE_STRUCTURE.md** | Обзор структуры курса, список практикумов, целевая аудитория | Студентам и преподавателям |
| **HPC_SETUP.md** | Как подготовить HPC-систему для проведения практикумов | Преподавателям (IT администраторам) |
| **TROUBLESHOOTING_COMMON.md** | Общие проблемы и решения, актуальные для всех модулей | Студентам и преподавателям |

## Быстрый старт

### Студентам

1. Прочитайте **COURSE_STRUCTURE.md** для понимания структуры курса
2. Выберите интересующий вас практикум:
   - `genome_annotation/` — Аннотация геномов
   - `variant_calling/` — Поиск геномных вариантов
3. Следуйте инструкциям в **README.md** выбранного практикума

### Преподавателям / Администраторам

1. Прочитайте **HPC_SETUP.md** для подготовки системы
2. Для каждого практикума см. **INSTRUCTOR.md** в его папке
3. При проблемах консультируйтесь с **TROUBLESHOOTING_COMMON.md**

## Часто задаваемые вопросы

**Q: Что будет добавлено в shared/ в будущем?**

A: По мере развития курса планируется добавить:
- Общие скрипты для управления всеми окружениями
- Единую базу данных B. subtilis 168 (вместо дублирования в каждом модуле)
- Больше shared утилит для валидации и отладки

Пока что каждый практикум полностью независим.

**Q: Почему данные скачиваются дважды?**

A: Каждый практикум может использоваться независимо, поэтому он должен быть самодостаточным. Размер B. subtilis 168 (~4 МБ) и его индексов небольшой.

**Q: Могу ли я запустить оба практикума одновременно?**

A: Да, они полностью независимы и используют разные conda-окружения (`ngs-annotation` и `ngs-variants`).
